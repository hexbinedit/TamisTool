# TamisTool Report Guide

> How to read and use TamisTool reports. Intended to be shareable with clients or junior analysts
> who may receive a report but have not used the tool.

---

## What is a TamisTool Report?

A TamisTool report is a point-in-time snapshot of the data available in a Microsoft Sentinel or
Defender workspace. It answers the question: **"What can we actually investigate in this environment?"**

Reports are generated by the analyst after running a scan and are designed to serve two audiences:

- **Internal analysts** — rapid visibility assessment before beginning an investigation
- **Client stakeholders** — a plain-language summary of SIEM coverage and identified gaps

---

## Report Sections

### Executive Summary

A high-level overview with four key numbers:

| Metric | What it means |
|---|---|
| Tables Discovered | Total number of data tables found in the workspace |
| Active Tables | Tables with data ingested in the past 30 days (configurable) |
| Empty Tables | Tables present but containing no rows |
| Baseline Coverage % | Percentage of well-known security tables that are present and active |

A coverage percentage **above 80%** generally indicates a well-instrumented environment.
A percentage **below 50%** suggests significant visibility gaps.

---

### Table Inventory

A full list of every table found in the workspace, with:

- **Status** — `healthy`, `empty`, `stale`, `missing`, or `warning` (see below)
- **Row Count** — approximate number of records
- **Last Event** — the most recent timestamp seen in the table

#### Status Definitions

| Status | Meaning | Action |
|---|---|---|
| `healthy` | Active data, within expected recency | No action needed |
| `empty` | Table present but no rows | Investigate connector configuration |
| `stale` | Has data but no recent ingestion | Check connector health or licensing |
| `missing` | Not present in the workspace at all | Connector not deployed |
| `warning` | Present but unusually low volume | Review parsing or connector settings |

---

### Baseline Gap Analysis

A comparison of discovered tables against TamisTool's catalog of well-known Sentinel and Defender
tables. Each entry shows the coverage status and which investigative use cases are affected by
a gap.

**What to look for:**

- **Red (missing)** entries indicate data sources that are entirely absent — investigations
  dependent on these tables will have no visibility
- **Yellow (empty/stale)** entries indicate connectors are deployed but not functioning correctly
- **Green (present)** entries are confirmed available for investigations

---

### Investigator Readiness

A use-case-oriented view that maps common investigation types to the tables they require.

| Use Case | Example tables required |
|---|---|
| Identity investigation | SignInLogs ✓, AuditLogs ✓, SecurityEvent |
| Endpoint investigation | DeviceEvents ✓, DeviceProcessEvents ✓, DeviceNetworkEvents ✓ |
| Network investigation | CommonSecurityLog, DnsEvents, WindowsFirewall |
| Cloud investigation | AzureActivity ✓, AzureDiagnostics |

A ✓ indicates the table is present and active. A ✗ indicates a gap. This section is designed
to give a rapid answer before an investigation begins.

## Report Formats

| Format | Best for |
|---|---|
| HTML (default) | Sharing with clients, browser-based review |
| CSV | Importing into Excel or Google Sheets for further analysis |
| JSON | Feeding into dashboards, ticketing systems, or pipelines |
| Parquet | Data engineering and long-term trend analysis |

---

## How to Generate a Report

```bash
# Generate HTML report from latest scan
tamis report generate --client acme-corp

# Generate CSV from a specific scan
tamis report generate --client acme-corp --scan-id 42 --format csv --output ./acme_42.csv

# Generate all formats
tamis report generate --client acme-corp --format html
tamis report generate --client acme-corp --format csv
tamis report generate --client acme-corp --format json
```

---

## Sharing Reports with Clients

The HTML report is self-contained and can be shared directly as a file attachment. It does **not**
contain any credentials, workspace IDs, or other sensitive configuration data — only the scan
results.

For client-facing use, consider reviewing the baseline gap analysis section before sharing to
ensure the framing is appropriate for the audience.
